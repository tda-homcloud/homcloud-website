Analysis of a 3D pointcloud¶

This tutorial explains:

  1. How to compute a persistence diagram (PD) from a given a 3D pointcloud data
  2. How to visualize the PD
  3. How to extract each birth/death times
  4. How to see the geometric origin of a birth-death pair

Since these analyses are common for any pointcloud data, this tutorial is helpful if you want to any kinds of pointclouds.

Computing a PD¶

The input data for this tutorial is sotred in pointcloud.txt. This file have 1000 points in a 3D space.

Now we try the analysis of the data.

First of all, we import some libraries.

In [1]:
import homcloud.interface as hc  # HomCloud
import homcloud.paraview_interface as pv  # 3
import numpy as np
import matplotlib.pyplot as plt
%matplotlib inline

Load the data by np.loadtxt.

In [2]:
pointcloud = np.loadtxt("pointcloud.txt")

Try to see the information of the pointcloud.

In [3]:
pointcloud.shape
Out[3]:
(1000, 3)

The shape of the array is 1000x3, this means that the number of points is 1000, and each point is localed in the 3D Euclidean space.

In [4]:
np.min(pointcloud, axis=0), np.max(pointcloud, axis=0)
Out[4]:
(array([0.00026274, 0.00100671, 0.00020878]),
 array([0.99997803, 0.9993682 , 0.99825316]))

The points are distributed in the cube $[0, 1]\times[0, 1]\times[0, 1]$. Now we visualize the pointcloud by paraview.

In [5]:
pv.show([pv.PointCloud.from_array(pointcloud)])

Now we compute a PD from the data. We use hc.PDList.from_alpha_filtration for that purpose. save_boundary_map option argument is required to compute optimal volumes later.

In [6]:
hc.PDList.from_alpha_filtration(pointcloud, 
                                save_to="pointcloud.idiagram",
                                save_boundary_map=True)
Out[6]:
PDList(path=pointcloud.idiagram)

Then the file pointcloud.idiagram is created. This file contains all information about the PD.

Visualize a PD¶

Next, we plot the PD.

Load pointcloud.idiagram by hc.PDList.

In [7]:
pdlist = hc.PDList("pointcloud.idiagram")

The returned object has PDs of all degrees. We get the 1st PD by dth_diagram.

In [8]:
pd1 = pdlist.dth_diagram(1)

Then we construct a histogram from the 1st PD and plot the histogram. histogram method construct a histogram, then plot it by plot method.

In [9]:
pd1.histogram().plot()

This figure shows something at the bottom-left. This is because many birth-death pairs are concentrated around there.

Therefore, we change the colorbar from linear-scale to log-scale.

In [10]:
pd1.histogram().plot(colorbar={"type": "log"})

In 1st PDs, a birth time with a large lifetime (death - birth) corresponds a "meaningful" ring. Therefore, the ring corresponding the birth-death pair (0.0025, 0.008) is, in some sense, the most significant ring.

Here, we remark that HomCloud uses the squares of radii in the birth and death axes.
For example, for the birth-death pair (0.5, 0.7), the ring appear at radius $\sqrt{0.0025}=0.5$ and disappears at $\sqrt{0.008} \approx 0.09$. This is maily because of the internal implementation of HomCloud, but this is also because squares are better for weighted pointclouds (pointclouds with initial radii).

If you do not want to use squares, please use the argument no_squared=True when hc.PDList.from_alpha_filtration is called.

Next we investigate the PD around (0.005, 0.005). HomCloud automatically adjusts the range of a histogram to plot all birth-death pairs. We can change the range of a histogram by x_range argument of histogram method.

In [11]:
pd1.histogram((0, 0.01)).plot(colorbar={"type": "log"})

Of course, there is no meaningful information in the PD since the data is randomly generated. However, it is very helpful for you to know typical PDs for random data.

If you want to change the resolution of the histogram, please use x_bins argument of histogram method. The default value of x_bins 128, and this means that the resolution of histogram is 128x128. We change the resolution into 256x256.

In [12]:
pd1.histogram((0, 0.01), 256).plot(colorbar={"type": "log"})

These figures are plotted by using matplotlib, so we can save the figure by matplotlib.pyplot.savefig. The following code saves the figure into pointcloud-pd1.png.

In [13]:
pd1.histogram((0, 0.01), 256).plot(colorbar={"type": "log"})
plt.savefig("pointcloud-pd1.png")

The 2 dim PD is also visualized.

In [14]:
pd2 = pdlist.dth_diagram(2)
In [15]:
pd2.histogram().plot(colorbar={"type": "log"})

In the PD, there are some birth-death pairs whose birth time is larger than 0.0125 with large lifetimes (death times minus birth times).

Inverse analysis of PD¶

Each birth-death pair in a PD corresponds to a topological structure such as a ring and a cavity. But it is not easy to find such a structure. Finding such a structure is called "inverse analysis".

Here, a powerful inverse analysis called "optimal volume" is introduced. Now we identify the ring structure corresponding to the birth-death pair (0.5, 0.7).

If you want to know about optimal volumes in details, please see the paper by Obayashi.

By using pd.nearest_pair_to, we pick up the birth-death pair nearest to (0.5, 0,7).

In [20]:
pair = pd1.nearest_pair_to(0.0025, 0.008)

This object is an instance of the class Pair, and the object has birth time and death time.

In [21]:
pair
Out[21]:
Pair(0.0025623095880009357, 0.008135892057712315)

The following code computes optimal volume.

In [22]:
optimal_volume = pair.optimal_volume()

boundary_points shows the coordinates of points in the ring structure.

Now we try to visualize multiple optimal volumes. We focus on the birth-death pairs whose birth times are less than 0.005 and death times are larger than 0.006.

In [26]:
pairs = [pair for pair in pd1.pairs() if pair.birth_time() < 0.004 and pair.death_time() > 0.006]
In [27]:
len(pairs)
Out[27]:
54

The number of pairs is 54. The optimal volumes for these pairs are computed. Since the number is a bit large, the computation time is quite long. To reduce the time, we use cutoff_radius parameter of optimal_volume method. This parameter restricts the simplices used in the computation. The simplices are restricted in the sphere whose radius is the given value. Now cutoff_radius is 0.5. This radius is estimated by the half of the length of the side of the cube.

In [28]:
%%time
optimal_volumes = [pair.optimal_volume(cutoff_radius=0.5) for pair in pairs]

CPU times: user 37.3 s, sys: 953 ms, total: 38.2 s
Wall time: 44.9 s
In [29]:
pv.show(
    [v.to_pvnode(gui_name="Volume for ({:.5f}, {:.5f})".format(v.birth_time(), v.death_time())) for v in optimal_volumes] 
    + [pv.PointCloud.from_array(pointcloud, gui_name="Point cloud")],
    wait=False
)

Inverse analysis for 2 dim PD¶

We also visualize the voids in the same way. We focus on pairs whose birth time is larger than 0.0125 and whose lifetimes are larger than 0.0005.

In [30]:
pairs = [pair for pair in pd2.pairs() if pair.birth_time() > 0.0125 and pair.lifetime() > 0.0005]
In [31]:
optimal_volumes = [pair.optimal_volume() for pair in pairs]
In [32]:
pv.show(optimal_volumes + [pv.PointCloud.from_array(pointcloud, gui_name="Pointcloud")], wait=False)

We can find some large voids in the pointcloud.
