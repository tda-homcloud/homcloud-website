結晶データの解析、特にvolume-optimal cycleとtightened-volumeについて¶

この文章では3x3の立方格子のデータを解析します。 完全な立方格子ではなく、ノイズが加えたデータを解析します。ここでは1次のパーシステント図を解析します。

まず、パーシステント図を計算します。

In [13]:
python3 -m homcloud.pc_alpha -D -I -d 3 --no-square cubic_crystal.txt cubic_crystal.idiagram
In [14]:
python3 -m homcloud.plot_PD -d 1 -x 0.4:0.75 -l cubic_crystal.idiagram -o cubic_crystal_PD1.png
display < cubic_crystal_PD1.png




































birth が0.5, death 0.7のあたりに点が集中しています。これは立方格子内の一辺の長さ1の正方形が作るリングが対応しています。


birth は 1 の 1/2 の 0.5、death は $\sqrt{2}$ (正方形の対角線の長さ) の 1/2 の 0.70.. が対応しています。ノイズを加えている分だけ
birth-death pairが拡散しています。






















次にこのbirth-death pairに対応するリングをvolume-optimal cycle (optimal volume)を使って計算しましょう。まずすべての
birth-death pairを表示します。

6頂点からなる長方形状のものが計算されていることがわかります。このbirth-death pairは一辺の長さ1の正方形と対応する、という話ですがこの
場合は長方形が得られます。実はパーシステントホモロジーでは複数の単位構造がくっついたものが現れることがよくあります。


ここで、この長方形はどれほど「あてになる」ものなのでしょう。つまりノイズに対しどれほど頑健なのでしょう。実のところこういった
「複数の単位構造がくっついたもの」はノイズに対して不安定です。


では、何か「ノイズに対しもっと安定なもの」がないでしょうか。HomCloudにはtightened volumeというのが実装されていて、それが使えます。






















それでは tightened volume を計算しましょう。 これは homcloud.optimal_volume に --tightened-volume THRESHOLD というオプションを与えると
計算できます。


















In [42]:






# 可視化
python3 -m homcloud.optimal_volume -d 1 -x 0.5091621008021502 -y 0.6975775524547941 --tightened-volume 0.05 \
  --no-optimal-volume on -P cubic_crystal.idiagram

Note¶
optimal volume というのは volume-optimal cycleの内側の領域のことです。実際のところ計算しているのは
optimal volumeでvolume-optimal cycleはその境界を取っています。


--tightened-volume の引数の 0.05 というのは、およそ「このくらいのノイズを加えても安定な情報を調べる」ということを
意味しています。ここでは(0.5, 0.7)近辺のbirth-daeth pairの分布範囲を目安に値を決めています。


ここで使った tightened volume というものが本当に「安定な情報」を見ているのか、というのは
今のところ数学的な保証はありません。おそらく何らかの数学的な特徴付けが可能だと思いますがそれは今後の課題です。


--tightened-volume の他に --tightened-subvolume, --owned-volume という似たようか機能があります。
計算アルゴリズムが違うのですが得られる結果は似ています。これらはvolume-optimal cycleの計算結果を使うので
-no-optimal-volume on は外してください。
